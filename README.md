# Card Payment Service

**REST-сервис для перевода денежных средств между банковскими картами.**

Проект реализован в рамках курсовой работы. Архитектура включает в себя встроенный фронтенд для тестирования, систему логирования и мониторинг с помощью связки **Prometheus + Grafana**. Для удобства развертывания и масштабирования используется контейнеризация через **Docker Compose**.

---

## 📦 Функциональность

*   **Инициирование перевода:** Создание запроса на перевод средств с одной карты на другую.
*   **Подтверждение перевода:** Валидация операции с помощью 4-значного кода подтверждения.
*   **Логирование:** Запись всех операций и событий в файл для последующего аудита.
*   **Мониторинг:** Сбор метрик производительности и состояния приложения для Prometheus.
*   **Встроенный фронтенд:** Одностраничное приложение (SPA) для удобного ручного тестирования API.

---

## ⚙️ Технологии

*   **Язык:** Java 21
*   **Фреймворк:** Spring Boot 3.5.13
*   **Сборка:** Gradle
*   **База данных:** Встроенная H2 (эмулирует PostgreSQL).
*   **Контейнеризация:** Docker / Docker Compose
*   **Мониторинг:** Prometheus, Grafana
*   **Тестирование:** Юнит и Интеграционное

## 🚀 Инструкция по запуску

### 1. Предварительные требования

Для работы проекта на вашей машине должны быть установлены:
*   **Docker Desktop** версии 27+ (или Docker Engine с активированным плагином Compose V2).
*   **Git** (для клонирования репозитория).
*   Свободная оперативная память: не менее 4 ГБ.

### 2. Запуск проекта

Выполните следующие команды в терминале:

```bash
# 1. Клонируйте репозиторий на свой компьютер
git clone <url-репозитория>
cd money-transfer-service/deploy

# 2. Соберите образы и запустите все сервисы в фоновом режиме (detached mode)
docker compose up -d --build

# 3. Проверьте, что все контейнеры запустились успешно
docker compose ps
```

**Важное замечание о базе данных:**
В качестве базы данных используется легковесная файловая СУБД **H2**. Для совместимости с кодом приложения она работает в режиме эмуляции PostgreSQL. Все данные (транзакции, карты) сохраняются в файл внутри контейнера, который монтируется в Docker как постоянное хранилище (`volumes`). Это гарантирует сохранность данных при перезапуске контейнеров.

### 3. Адреса сервисов

После запуска вы можете получить доступ к приложению и панелям мониторинга по следующим адресам:

| Сервис | Адрес (локальный) | Описание |
| :--- | :--- | :--- |
| **API + Фронтенд** | `http://localhost:5500` | Основной адрес приложения. Здесь доступен встроенный фронтенд для тестирования. |
| **Prometheus** | `http://localhost:9090` | База данных временных рядов для метрик вашего приложения. |
| **Grafana** | `http://localhost:3000` | Визуализация данных из Prometheus (логин: `admin`, пароль: `admin`). |

> **Важно:** Встроенный фронтенд (SPA) упакован вместе с бэкендом и автоматически доступен по порту `5500`. При переходе на этот адрес вы сразу увидите пользовательский интерфейс для работы с переводами.

## 📡 Примеры запросов к API

Вы можете использовать `curl` или Postman для взаимодействия с REST API.

### 1. Инициировать перевод

Отправьте `POST`-запрос на создание нового перевода.
```bash
curl -X POST http://localhost:5500/transfer \
  -H "Content-Type: application/json" \
  -d '{
    "cardFromNumber": "1111111111111111",
    "cardFromValidTill": "12/27",
    "cardFromCVV": "123",
    "cardToNumber": "2222222222222222",
    "amount": {"value": 1000, "currency": "RUB"}
  }'
```
В ответ вы получите объект перевода, содержащий уникальный идентификатор (`id`), который понадобится для подтверждения.

### 2. Подтвердить перевод

Используйте полученный `id` и код подтверждения (`code`), чтобы завершить операцию.
```bash
curl -X POST http://localhost:5500/transfer/confirm \
  -H "Content-Type: application/json" \
  -d '{
    "transferId": "<ID_ИЗ_ПРЕДЫДУЩЕГО_ЗАПРОСА>",
    "code": "4815"
  }'
```
При успешном подтверждении статус перевода изменится на `SUCCESS`.
